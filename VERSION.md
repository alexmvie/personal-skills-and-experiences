## Versioning Rules

### When to increment MAJOR (x.0.0)
- New major section added (e.g., "Publikationen", "Open Source")
- Complete restructuring of existing sections
- Fundamental changes to portfolio structure
- New file format or deployment system

### When to increment MINOR (0.x.0)
- New skill/technology added
- New project showcased
- Expanded existing section with substantial content
- New feature in workflow/documentation

### When to increment PATCH (0.0.x)
- Typo/spelling correction
- Version number update
- Accuracy fix (e.g., GPT-5-Codec → GPT-5-Codex)
- Minor content refinement
- Footer/date updates

**Deployment Checklist:**
